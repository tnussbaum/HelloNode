So ... Node.js

Node.js

  • Created in 2009 by Ryan Dahl (Joyent)
  • V8 Javascript engine, an event loop (libuv underneath), and an I/O API
  • First on Linux, then other Unix, then ported to Windows
  • Since its introduction, it has contributed to the popularity of Javascript
    • Ranks 8th, 2.3% on TIOBE March 2016 index

Why so popular?

  • Ease of use / approachability
  • Emphasis on I/O performance
  • Focus in modular development

Popularity - Ease of use

  • One language, client / server
  • Large developer base
  • Simple servers, easy to get started

Popularity - Performance

  • Asynchronous, yet single threaded
    • "Everything runs in parallel, except your code"
    • Ruby EventMachine, Python Twisted
  • Works extremely well when applications are I/O-bound
  • Event-based programming (natural for Javascript)

Popularity - Modularity

  • Solve problems in small pieces
  • Similar to Unix command line approach
  • Leads to "throw-away-ability"

What makes Node.js great?

  • NPM / ecosystem
    • Package/library management
    • Lots of problems already solved
  • Node.js / Javascript tools
    • Grunt, Gulp, etc.
    • IDE and text editor support
      • Atom
      • Visual Studio Code

Combining Node.js / Java

  • Avatar.js
  • Nodyn
  • Trireme

Avatar.js

  • java.net open source project from Oracle
  • Brings Node.js "inside" the JVM
  • Built on (requires) Java8
    • Utilizes invokedynamic and Nashorn Javascript engine
  • JNI wrappers around libuv
  • Bundles patched version of Node.js APIs
Demos!

Avatar.js Issues / Future

Issues

  • Performance
  • Compatibility - worst of the 3 solutions
  • Dependencies

Project Avatar

  • BaaS solution based on Avatar.js and Glassfish

Nodyn

  • Open source project from JBoss/Redhat (TorqueBox)
  • Implementation of Node.js in Java (7+) / DynJS
  • Leverages Netty and vert.x
  • Node.js API
    • Ported to Java
    • or wrapped via JNI
  • Can be run stand-alone, or embedded in larger Java app
More demos!

Nodyn Issues / Future

Issues

  • Performance (mainly in DynJS)
  • Compatibility - mostly compatible with Node.js 0.10.x
  • Dependencies

Future

  • Uncertain, but also not tied to Oracle

Trireme

  • Github project from Apigee (API monitoring app)
  • Similar to Nodyn - Node.js re-implemented in the JVM
  • Rewrites some behaviors of Node.js to leverage Java threading
  • Uses Rhino engine, with Java implementations of some Node.js modules
  • Designed for embedding in a larger Java application
More demos!

Trireme Issues / Future

Issues

  • Performance (Rhino, JVM startup time)
  • Compatibility - best compatibility with Node.js 0.10.x
  • Dependencies

Future

  • Promising - new version with upgrade to Nashorn, newer Node.js support
But why?

Why Node.js+Java?

  • Java has solved a lot of business problems
  • Businesses have invested in infrastructure
  • The JVM is fast
    • JIT
    • Threading
    • Scalability
  • Node.js / Javascript efficiencies with newer app designs

What's next?

  • 3 different approaches to the same problem
  • None of them are "enterprise-ready" solutions, but can fill niche needs
  • Many opportunities to learn, not just with Javascript
  • Trireme is in the best position to progress, specifically in servlet implementation
